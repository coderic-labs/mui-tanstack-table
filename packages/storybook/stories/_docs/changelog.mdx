# Changelog

## 0.0.7

- Froze a validated release baseline for peer dependencies to make setup and support reproducible.
- Confirmed compatibility across React 18/19 and MUI 6/7 combinations.
- Confirmed compatibility with `@tanstack/react-table` v8 and `react-intl` v7 combinations used in demos.

### Validated Baseline

- `react` 19.2.5
- `react-dom` 19.2.5
- `@mui/material` 7.3.10
- `@mui/icons-material` 7.3.10
- `@mui/x-date-pickers` 7.29.4
- `@tanstack/react-table` 8.21.3
- `@emotion/react` 11.14.0
- `@emotion/styled` 11.14.1
- `react-intl` 7.1.14

### Upgrade Notes

- If you are upgrading from earlier `0.0.x` versions, prefer the validated baseline above first, then adjust peers as needed.
- Peer dependency ranges are unchanged; this release documents tested combinations and establishes a stable support anchor.

## 0.0.6

- Implemented multi-column pinning for left and right table edges.
- Column pinning now supports any number of pinned columns (not limited to first/last).
- Users can pin/unpin columns via the header options menu.
- Added comprehensive Cypress test coverage for column pinning scenarios:
  - Multi-column pinning interactions
  - Pinned column visibility toggling
  - Pinned column persistence across pagination and dataset changes
- Improved server-side table demo with footer content for better test coverage.

### Upgrade Notes

- Column pinning is fully controlled by TanStack Table state; users interact via header options menu.
- Pinned columns automatically adjust their position when visibility changes or data is paginated.

## 0.0.5

- Added row highlighting and cell style customization support through table rendering props.
- Improved table cell rendering internals by introducing a dedicated table cell component.
- Refined row-selection behavior to use stable string row IDs across table interactions.
- Expanded Cypress component coverage for pagination, row detail, row selection, and cross-page bulk deletion flows.

### Upgrade Notes

- If you were using row-level styling APIs from previous pre-release builds, migrate to the current cell styling approach (`getCellStyle`).
- Selection state keys are now normalized as string row IDs; custom integrations that inspect `rowSelection` directly should treat keys as strings.

## 0.0.4

- Initial public alpha release for the Storybook package.
- Added MUI table rendering components and toolbar primitives.
- Demonstrated client-side and server-side table examples.
- Included row selection, expansion, filtering, sorting, pagination, and column visibility toggles.
- Built with MUI v6 and TanStack Table v8.

## Notes

This library is still evolving. Expect API changes and additional documentation as the package matures.

> If you want to help shape the roadmap, please raise issues or feature requests in the repository.
