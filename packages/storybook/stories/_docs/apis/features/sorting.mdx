# Sorting API

Sorting uses the same header UI across both modes, but the place where rows are ordered is different. Choose one approach per table.

## Client side approach

Use TanStack client row sorting.

- Set `sortingFn` per sortable column when needed.
- Include `getSortedRowModel()`.
- Optional multi-sort: `enableMultiSort` and `maxMultiSortColCount`.

<Source code={SortingClientSideDemoRaw} language="tsx" dark />

## Server side approach

Use controlled sorting state and map it to your backend query.

- Enable `manualSorting: true`.
- Pass `state.sorting` and `onSortingChange`.
- Fetch sorted rows externally and return the current page payload.
