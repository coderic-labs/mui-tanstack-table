# Confirm Modal

Confirm dialogs are essential for protecting users from accidental destructive actions like deletion. The `useConfirmDialog` hook provides a flexible pattern for integrating custom confirmation dialogs into your table workflows.

## How it works

The `useConfirmDialog` hook manages dialog state and connects your custom dialog component to confirmation handlers:

- **Custom Dialog Component**: Create a Material-UI Dialog component that accepts `ConfirmDialogProps<TData>`.
- **Confirmation Handler**: Define what happens when the user confirms the action.
- **Show Dialog**: Call `showConfirmDialog(data)` to open the dialog with specific data.
- **Auto-close**: Dialog closes automatically after confirmation or user cancellation.

### ConfirmDialogProps interface

Your custom dialog component must accept the following props:

```tsx
type ConfirmDialogProps<TData> = {
    open: boolean;           // Dialog open/closed state
    data: TData;             // Data passed to showConfirmDialog()
    onClose: () => void;     // Called when user cancels
    onConfirm: () => void;   // Called when user confirms
}
```

### Generic data support

The hook works with any data type. Common patterns:

- **Single IDs/objects**: `useConfirmDialog<string | Developer>`
- **Array of IDs**: `useConfirmDialog<string[]>`
- **Complex objects**: `useConfirmDialog<{ action: string; data: Developer }>`

The `TData` type flows through to your dialog component via `ConfirmDialogProps<TData>`.
