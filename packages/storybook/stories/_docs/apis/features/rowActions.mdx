# Row Actions

Row actions let users trigger item-level operations (for example edit or delete) directly from each row.

## Shared approach

- Add a display column (commonly `id: 'actions'`) for action controls.
- Render row-level buttons in `cell` using `IconButton` or `Button`.
- Handle actions directly in the cell renderer (the demo uses `alert` for edit/delete).
- Use table meta only when handlers need shared table-level wiring.
- Keep actions row-scoped: each handler should target `row.original` data.
